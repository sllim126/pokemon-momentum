Profiling Notes (April 2026)
----------------------------

- First-load profiling pass completed on the current dashboard boot path.
  - Container timing sample for the default card dashboard path:
    - `/series?productId=88029&subTypeName=Reverse+Holofoil&days=365&category_id=3` ~= 472 ms
    - `/health?category_id=3` ~= 227 ms
    - `/top_movers?days=30&limit=120&min_prior=5&product_kind=card&category_id=3` ~= 169 ms
    - `/sma30_holds?...` ~= 121 ms
    - `/tracking/auth_config` and `/categories` were negligible by comparison
  - Browser waterfall on a URL-selected `early_uptrends` view showed slower first paint requests:
    - `/early_uptrends?...` ~= 2.19 s
    - `/series?...` ~= 1.55 s
    - `/universe?limit=1...` ~= 1.35 s
    - `/health?category_id=3` ~= 1.27 s
- Current first-load suspects worth reducing next:
  - avoid first-paint dependence on `/health`
  - avoid metadata fallback hits like `/universe?limit=1...` when a selected row already has enough payload
  - keep heavier non-default table endpoints (`early_uptrends`, `group_products`, etc.) from blocking the first usable screen when they come from URL state
